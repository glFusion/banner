glFusion Banner Plugin Configuration

The Banner Plugin allows you to create and display banner ads. Banners may be displayed within the header and footer templates, as well as in blocks, stories, and static pages.

Main Banner Options

Administrator's Email Address
Enter the email addresses to which confirmation email messages should be sent. If this is empty, the site email address will be used.
Display banners in templates?
Select whether to display banners in templates. If "No", then no banners will appear in the header or footer templates.

IMPORTANT: If you enable this function, and you have categories enabled which use any of the standard template names (header, footer, article, etc.), then you MUST make sure that the template variables are defined properly in order to keep your impression count accurate.

The plugin has no way of knowing whether template variables are actually defined, so it counts each attempt to display an ad whether it is actually displayed or not.

Enable Submission Queue?
Setting this to "Yes" enables the submission queue. If this is "No", the submission queue is not used.
Allow submissions from site members?
Setting this to "Yes" allows site members to submit ads, subject to the "Enable Submission Queue" setting.
Notification Email?
Set this to "Yes" to have an email message sent to the site admin when a new banner is submitted.
Delete Banner with Owner?
Set this to "Yes" to have banners deleted when the owner's user account is deleted. If this is "No", banners will persist after the owner has been removed, and the ownership will change to a member with the "banner.admin" privilege.
Show baners on admin pages?
Select whether to display banners on pages under the site admin URL. If impressions are counted, this should probably be set to "No".
Show target links in a new window?
If this is "Yes", then clicking on a banner will open the target in a new browser window or tab. If this is "No", then the target will be opened in the current window, effectively leaving your site.
Image Max Height
Enter the maximum image height for an uploaded banner image, in pixels.
Image Max Width
Enter the maximum image width for an uploaded banner image, in pixels.
Default Weight
Select the default weight to be applied to new ads. The weight indicates the liklihood that a banner will be displayed relative to other banners.

Banner Display Settings

These settings allow you to block ads from being displayed to various classes of visitor. Some ad providers, such as Google Adsense, have clauses in their TOS regarding invalid or webmaster-generated impressions. Setting one or more of the following items can help mitigate this problem.

Show Banners on Admin Pages?
If this is "Yes", then banners will be displayed on pages under the site admin URL. If "No", then admin pages will not be shown banners. This is one method to reduce false impressions due to admin activity.
IP addresses that will not be shown ads
Enter one or more IP addresses that will not be shown ads.
User Agents that will not be shown ads
Enter one or more User Agent strings that will not be shown ads. You can enter a substring, e.g. "Mozilla"e; to catch all versions of Mozilla browsers. The user-agent string is not case-sensitive. This is typically used to prevent monitoring systems (Nagios, Xymon, etc.) from increasing the ad hit counts.
Show ads to the ad owner?
Set this to "No" to hide ads from their owner. If this is "Yes", then the ad owners will see their own ads. Since the impression and click counters can be controlled by other settings, this is most useful if your site uses Adsense or similar ads to prevent false impression counts.
Show ads to ad administrators?
Set this to "No" to hide all ads from the banner ad administrators (members with the 'banner.admin' privilege. For these users, this "No" setting will override a "Yes" setting for "Show ads to the ad owner". If this is "Yes", then the administrators will see ads. Set "Show ads to the ad owner" to "No" to prevent administrators from seeing their own ads. Since the impression and click counters can be controlled by other settings, this is most useful if your site uses Adsense or similar ads to prevent false impression counts.
Count ad clicks made by the ad owner?
If this is set to "No", then clicks made by the ad owners will not count towards their hit counts. If this is "Yes", then clicks made by ad owners will be added to the ad's hit count. This almost certainly should be set to "No".
Count ad clicks made by ad administrators?
If this is set to "No", then clicks made by the ad administrators will not count towards their hit counts. If this is "Yes", then clicks made by administrators will be added to the ad's hit count. This probably should be set to "No".
Count ad impressions for the ad owner?
If this is set to "No", then the ad impression count is not incremented when the ad is displayed to its owner. If this is "Yes", then the display will be added to the ad's impression count. This almost certainly should be set to "No".
Count ad impressions for ad administrators?
If this is set to "No", then the ad impression count is not incremented when the ad is displayed to an ad administrator. If this is "Yes", then the display will be added to the ad's impression count. This probably should be set to "No".
Centerblock Enabled?
If this is set to "Yes", then ads which belong to categories that have "Show in Centerblock" set to "true" will be displayed in a center block according to the "Centerblock Position" setting. Setting this to "No" suppresses the banner ad centerbock.
Centerblock Position?
Select the position where the center block should appear. This has no effect unless the centerblock is enabled.
Centerblock Replaces Home Page?
If this is set to "Yes", then the site home page will be replaced by a single banner ad. Note that this may conflict with other plugins, so all other plugins should have their centerblock turned off if this is to be used.
Max. ads to show in blocks
This is the maximum number of ads to show in blocks. Enter "0" for no limit. The default random ad block is limited to a single ad, but other blocks may display one or more ads.
